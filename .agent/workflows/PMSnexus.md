# PMSnexus — Lateral Synthesis Engine

> Run when stuck on a hard architectural, scaling, or logic problem.
> Forces the AI to use cross-disciplinary mental models (biology, physics, economics) to find novel solutions.

---

## Step 1: The First Principles Deconstruction

```text
"Chairman has invoked /PMSnexus. The sticky problem is: [Chairman's input].

You MUST use `<thinking>` tags for Step 1 and Step 2. Do not output anything until Step 3.

Inside `<thinking>`:
1. Strip away all software, code, and IT terminology.
2. Define the absolute core physical, mathematical, or resource constraint. 
   (e.g., Instead of 'How to prevent API rate limits', write 'How to regulate resource consumption under external hostility')."
```

---

## Step 2: The Multiverse Mapping

```text
"Still inside `<thinking>`:
Force-map the core constraint to 3 STRICTLY UNRELATED scientific or systemic domains.
- FORBIDDEN DOMAINS: Software Engineering, IT, standard business management.
- REQUIRED DOMAINS: Biology, Physics, Thermodynamics, Game Theory, Astrophysics, Immunology, Mycology, Fluid Dynamics, etc.

For each of the 3 chosen domains, explicitly define how nature or science solves this EXACT constraint."
```

---

## Step 3: Architectural Translation & Output

```text
"Close the `<thinking>` tag. Present the synthesis to the Chairman.

Translate the abstract scientific concepts back into concrete code/architecture solutions for the current project. 
Format the output EXACTLY like this:

### 🧠 NEXUS SYNTHESIS

**Core Constraint (First Principles):** [1 sentence defining the stripped-down physical/logical problem]

**Option 1: The [Domain 1] Model — [Concept Name]**
- **The Science:** [1 sentence on how Nature/Science solves this]
- **The Architecture:** [Concrete algorithmic/code translation for our project]
- **Pros / Cons / Risk:** [Brief analysis]

**Option 2: The [Domain 2] Model — [Concept Name]**
- **The Science:** [...]
- **The Architecture:** [...]
- **Pros / Cons / Risk:** [...]

**Option 3: The [Domain 3] Model — [Concept Name]**
- **The Science:** [...]
- **The Architecture:** [...]
- **Pros / Cons / Risk:** [...]

Ask Chairman: 'Which mental model should we forge into an actual algorithm?'"
```
