# BSCdraft — Corporate Document Drafting

> "Cho tao 1 Tờ trình về việc X" → AG soạn đúng format BSC
> Learns templates from existing docs on SharePoint
> Cross-links: called by /BSCboard (TTr, NQ), standalone for any doc

---

## Step 1: Intake

```text
"Chairman has invoked /BSCdraft.

Supported document types:
| Code | Loại tài liệu | Template source |
|------|---------------|-----------------|
| BB   | Biên bản họp  | SharePoint BB cũ |
| TTr  | Tờ trình      | SharePoint TTr cũ |
| NQ   | Nghị quyết    | SharePoint NQ cũ |
| PLY  | Phiếu lấy ý kiến | SharePoint mẫu |
| BC   | Báo cáo       | SharePoint BC cũ |
| CV   | Công văn      | SharePoint CV cũ |

If called from another workflow (e.g., /BSCboard):
  - Accept context parameters: type, topic, related documents
  - Pre-fill from calling workflow's data

If called standalone:
  Chairman, specify:
  1. Document type (BB/TTr/NQ/PLY/BC/CV)
  2. Subject/topic
  3. Key points to include
  4. Recipients (if applicable)"
```

[HARD STOP] Wait for Chairman input (skip if context passed from calling workflow).

---

## Step 2: Draft

```text
"Based on Chairman's input:

1. SCAN SharePoint → find most similar existing document
   - If template found → apply format, headers, numbering, legal language
   - If NOT found:
     Option 1: Use generic BSC format → AG creates best-effort draft — Fast / May need heavy editing / Low risk
     Option 2: Ask Chairman for a reference file — Accurate / Slower / Low risk
     Option 3: Skip template, write from scratch — Flexible / Not standard / Medium risk

2. FILL content from Chairman's input + context from email/SharePoint
3. CROSS-CHECK: If TTr → check matching NQ draft consistency (numbers, names, dates)
4. ADD metadata footer:
   ---
   Generated by: AG (Antigravity AI Assistant)
   Date: [YYYY-MM-DD HH:MM]
   Source: [description]
   Reviewed by: SonPM ☐
   ---

5. SAVE draft to AG_Output/Drafts/
   Naming: [YYYYMMDD]_AG_[Type]_[Description]_v1.docx

Present draft to Chairman for review."
```

[HARD STOP] Chairman reviews and approves.

---

## Step 3: Finalize & Distribute

```text
"After Chairman approval:

1. Mark 'Reviewed by: SonPM ☑' in metadata
2. If sending required:
   - Draft email with document attached
   - Add signature: '[Drafted by AG — Reviewed & approved by SonPM]'
   - [HARD STOP] Chairman approves → send
3. MOVE draft to AG_Output/Archive/ (never delete)
4. If called from /BSCboard → return control to calling workflow"
```
